# Scrum 敏捷開發

## 3+5原則
- 三個角色
  - 產品負責人
    - 確定產品的樣子
    - 我？
  - 團隊成員
  - Scrum大師
    - Problem solver
    - 我？
- 五個流程
  - 衝刺規劃
    - 市場
    - 功能
  - 衝刺
    - 事情的優先次序
    - 時間一到兩週左右？（不太懂是指什麼的時間）
  - 每日Scrum立會
    - 每天花15分鐘，每個人報告工作進度與目標
  - 衝刺檢視
    - 每兩週給人試用，得到反饋
    - 給彥賢回報進度？
  - 衝刺回饋
    - 檢討會
    - 功能是否能滿足客戶的需求
